Manual de Ayuda

Bienvenido al manual de usuario del Sistema de Parqueaderos. Este sistema le permite gestionar la información de los coches que ingresan al parqueadero de forma sencilla y eficiente.

Funcionamiento del Sistema

El sistema permite gestionar los datos de los coches en el parqueadero de manera sencilla. A continuación, se explica cómo funciona:

1. Insertar Datos de un Coche

1. Insertar Datos de un Vehículo

Para registrar un vehículo en el parqueadero, el sistema le pedirá los siguientes datos:

  • Placa: El sistema le pedirá ingresar la placa del vehículo. Asegúrese de escribirla correctamente, siguiendo el formato adecuado.
  • Cédula: Ingrese la cédula del propietario del vehículo.
  • Nombre: Ingrese el nombre del propietario del vehículo.
  • Segundo Nombre: Ingrese el segundo nombre del propietario del vehículo.
  • Apellido: Ingrese el apellido del propietario del vehículo.
  • Segundo Apellido: Ingrese el segundo apellido del propietario del vehículo.
  • Fecha: La fecha de ingreso se generará automáticamente.
  • Hora de Ingreso: La hora de ingreso se generará automáticamente.

Una vez ingresados los datos, el sistema registrará el vehículo en el parqueadero. Si la placa ya está registrada en el parqueadero, el sistema le informará que el vehículo ya está parqueado y no permitirá el registro duplicado.

2. Uso del Árbol Binario de Búsqueda (BST)

El sistema utiliza un árbol binario de búsqueda (BST) para organizar y gestionar los datos de los vehículos. En un BST, cada nodo tiene un valor (en este caso, la placa del vehículo) y dos subárboles: uno a la izquierda y otro a la derecha.

El BST está organizado de la siguiente manera:

  • Los nodos a la izquierda tienen valores menores que el nodo padre.
  • Los nodos a la derecha tienen valores mayores que el nodo padre.

Esta estructura permite realizar búsquedas rápidas y eficientes, ya que se puede descartar la mitad de los nodos en cada paso de la búsqueda.

3. Almacenamiento y Recuperación de Datos

Los datos de los vehículos se almacenan en un archivo de texto en preorden. Esto significa que los datos se guardan en el orden en que se visitan los nodos del BST: primero el nodo padre, luego el subárbol izquierdo y finalmente el subárbol derecho.

Al leer los datos del archivo de texto, el sistema puede reconstruir el BST de manera eficiente, asegurando que los datos se mantengan organizados y que las búsquedas sigan siendo rápidas.

4. Buscar Vehículos

El sistema permite buscar vehículos registrados en el parqueadero utilizando diferentes criterios de búsqueda:

  • Buscar por Placa: Permite buscar un vehículo ingresando su placa.
  • Buscar por Cédula: Permite buscar un vehículo ingresando la cédula del propietario.
  • Buscar por Nombre: Permite buscar un vehículo ingresando el nombre del propietario.

5. Eliminar Vehículos

El sistema permite eliminar vehículos del parqueadero utilizando la placa del vehículo. Una vez eliminado, el vehículo ya no estará registrado en el parqueadero.

6. Visualizar Vehículos Parqueados

El sistema permite visualizar todos los vehículos actualmente parqueados en el parqueadero, mostrando la información relevante de cada vehículo.

Ejemplo de Uso:

        // Insertar un vehículo
        insertarVehiculo();

        // Buscar un vehículo por placa
        buscarPorPlaca("ABC1234");

        // Eliminar un vehículo por placa
        eliminarVehiculo("ABC1234");

        // Visualizar todos los vehículos parqueados
        visualizarVehiculosParqueados();

Esto significa que el sistema le pedirá cada dato uno por uno, y luego creará un registro con todos los datos del vehículo.

Funcionamiento de los Backups con Cobian Reflector

Para mantener la seguridad de sus datos y prevenir la pérdida de información valiosa, el sistema realiza automáticamente copias de seguridad de los archivos relacionados con el parqueadero. Esto incluye los archivos de texto donde se almacenan los datos de los coches registrados.

1. ¿Qué Archivos se Respaldan?

El sistema realiza copias de seguridad de los archivos .txt que contienen la información de los coches registrados en el parqueadero. Estos archivos son vitales para mantener la integridad de los datos, por lo que se respaldan de manera periódica.

2. Selección de los Archivos para el Backup

Cobian Reflector está configurado para seleccionar automáticamente los archivos relevantes del sistema. Los archivos de datos del parqueadero, típicamente almacenados con la extensión .txt, son los que se respaldan. Estos archivos pueden estar ubicados en la carpeta principal del sistema o en una subcarpeta designada para datos.

En el caso del sistema de parqueaderos, los archivos que contienen los registros de los coches se seleccionan automáticamente para ser respaldados, asegurando que siempre tenga una copia actualizada de los datos más recientes.

3. ¿Cómo se Crean los Backups?

El proceso de creación de backups con Cobian Reflector es automático. Cuando el sistema detecta que los archivos han cambiado o cuando se ha alcanzado el horario programado, Cobian Reflector comienza el proceso de copia de seguridad. Este proceso sigue estos pasos:

  1. Identificación de Archivos: Cobian Reflector identifica los archivos de datos relevantes (en este caso, los archivos .txt con los registros de los coches).
  2. Creación del Backup: Una vez identificados los archivos, Cobian Reflector realiza una copia exacta de estos archivos. Si los archivos ya existen en la carpeta de destino, se pueden sobrescribir o agregar una nueva versión, dependiendo de cómo esté configurado el sistema.
  3. Almacenamiento del Backup: La copia de seguridad se guarda en una ubicación segura que puede ser una carpeta en el disco duro, una unidad externa o incluso en una carpeta de red. Esto asegura que los archivos estén accesibles si se necesita restaurar la información.
  4. Notificación: En algunos casos, Cobian Reflector puede enviar una notificación indicando que el proceso de copia de seguridad se ha completado correctamente.

4. ¿Dónde se Guardan los Backups?

Los backups se guardan en una carpeta de destino que puede ser configurada durante la instalación del sistema. El destino más común es una carpeta local en su computadora, pero también puede configurar una unidad externa o una carpeta en red para almacenar las copias de seguridad, lo que proporciona una mayor seguridad ante fallos del sistema.

5. ¿Qué Hacer en Caso de Necesitar Restaurar los Datos?

Si alguna vez necesita recuperar la información de los coches registrados, simplemente abra Cobian Reflector, seleccione la tarea de backup correspondiente y elija la opción de restauración. El sistema reemplazará los archivos actuales con los datos de la copia de seguridad, asegurando que toda la información se recupere tal como estaba.
